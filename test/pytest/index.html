    pytest¶

    pytest是一款优秀的第三方单元测试框架，简单易用、功能强大：

    • 完全兼容与unittest（Python内置的单元测试框架）和nose（另一流行的第三方单元测试框架）
    • 测试用例失败时，诊断信息输出丰富合理，可以提高开发效率
    • 提供了大量功能强大的插件，如代码覆盖率、超时、PEP8代码风格检查等

    基本用法¶

    安装pytest¶

    1
    pipenv install --dev pytest

    运行上面命令，可以在虚拟环境里安装pytest包，并设置为开发环境的依赖。

    定义测试用例¶

    在pytest单元测试框架里，有如下约束：

    • 所有的单元测试.py源文件的名称都需要用test_开头或者_test结尾
    • 在单元测试源码里，可以包含test_开头的测试函数，也可以包含Test开头的测试类
    • 在单元测试类里，可以包含一个或多个test_开头的测试函数
    • 在测试函数里，使用基本的assert断言即可

        class TestClass:

        def test_one(self):

            assert 'h' in x

        def test_two(self):
            x = "hello"
            assert hasattr(x, 'check')

    所有nose和unittest框架下写好的测试用例，一般无需任何修改就可以被pytest识别和执行。

    fixture及其高级用法¶

    运行一些复杂测试时，我们可能需要先为测试准备好一些东西，如准备好数据或实例化特定的类等。传统的单元测试是用setup和teardown来实现的。在pytest里提供了简洁易用、功能强大的@pytest.fixture装饰器来避免大量的重复代码。下面的例子里我们定义了一个名为smtp的测试fixture，可以被test_ehlo和test_noop两个测试方法使用。

    import pytest

@pytest.fixture
def smtp():
    import smtplib
    return smtplib.SMTP("smtp.gmail.com", 587, timeout=5)

def test_ehlo(smtp):
    response, msg = smtp.ehlo()
    assert response == 250
    assert 0  # for demo purposes

def test_noop(smtp):
    response, msg = smtp.noop()
    assert response == 250
    assert 0  # for demo purposes

    有的时候测试结束时我们还需要对准备测试阶段创建出来的东西加以清理。通过在fixture函数里引入yield，就可以实现类似teardown的功能。比如下面的代码里，yield语句之后的部分就是需要测试清理的内容。

    import pytest
import smtplib

@pytest.fixture(scope="module")
def smtp(request):
    server = getattr(request.module, "smtpserver", "smtp.gmail.com")
    smtp = smtplib.SMTP(server)
    yield smtp
    print ("finalizing %s (%s)" % (smtp, server))
    smtp.close()

    在测试方法间共享fixture¶

    测试环境里所需的fixture，可在不同的范围内被一组测试方法所共享。这样可以避免多次初始化造成的额外开销。

    范围 功能
    function 每个test都运行，默认是function的scope
    class 每个class的所有test只运行一次
    module 每个module的所有test只运行一次
    session 不管在哪里定义都只运行一次

    数据驱动测试¶

    我们有时候希望能用若干组不同的数据来执行同一个测试函数，这时可用参数化的fixture来实现。

    1
2
3
4
5
6
7
8
    import pytest

@pytest.fixture(params=[0, 1], ids=["spam", "ham"])
def a(request):
    return request.param

def test_a(a):
    pass

    在上面例子里，虽然只定义了一个测试函数，但是实际执行时，会测试两个用例：test_a[spam]和test_a[ham]。

    临时测试路径¶

    有时测试函数并不需要直接访问一个fixture对象，比如测试需要用一个空的临时目录当作当前工作路径并做一些初始化工作，但并不关心当前具体路径。这时候，我们可以用tempfile来管理临时目录，并配合usefixtur装饰器来实现。

    import pytest
import tempfile
import os

# 创建一个临时目录作为当前工作目录，并装饰为一个fixture
@pytest.fixture()
def cleandir():
    newpath = tempfile.mkdtemp()
    os.chdir(newpath)

# 使用usefixture来装饰一个测试类，说明这个类里的所有测试方法都要先用cleandir这个fixture来做好测试准备
@pytest.mark.usefixtures("cleandir")
class TestDirectoryInit(object):

    # 测试函数，断言当前目录是空目录
    def test_cwd_starts_empty(self):
        assert os.listdir(os.getcwd()) == []
        with open("myfile", "w") as f:
            f.write("hello")

    * 你也可以用usefixtures指定多个测试准备用的fixture
* 你也可以用@pytest.fixture(autouse=True)代替usefixtures来自动指定fixture

```py3
import pytest

class DB(object):
    def __init__(self):
        self.intransaction = []
    def begin(self, name):
        self.intransaction.append(name)
    def rollback(self):
        self.intransaction.pop()

@pytest.fixture(scope="module")
def db():
    return DB()

class TestClass(object):
    @pytest.fixture(autouse=True)
    def transact(self, request, db):
        db.begin(request.function.__name__)
        yield
        db.rollback()

    def test_method1(self, db):
        assert db.intransaction == ["test_method1"]

    def test_method2(self, db):
        assert db.intransaction == ["test_method2"]
```
