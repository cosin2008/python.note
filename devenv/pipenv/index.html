    Pipenv¶

    Python开发鼓励开发者使用包的概念来复用已经写好的公有模块。这些包一般是通过Pypi来管理的。但开发不同的Python应用，显然需要依赖不同功能、不同版本的包，甚至可能需要在不同版本的CPython下执行。为了更好的解耦和定义开发环境对包的依赖，Python开发社区创造了虚拟环境的概念。较流行的实现就是为特定的Python开发项目克隆一份特定版本的Python执行环境作为虚拟环境，然后在这个虚拟Python执行环境里安装所依赖的包，而不去污染全局的Python执行环境。然而这样的话，开发者常常需要管理多个不同的Python开发项目和对应的虚拟环境，于是又有其它一些五花八门的工具被开发出来帮助开发者自动选择与当前工程匹配的虚拟环境等。开发者还需要在使用pip安装依赖包后，设法维护requirements.txt文件来记录新的依赖包，比较繁琐也容易遗漏。

    Pipenv实际上就是一个一揽子解决方案。它是由著名的Requests库的作者开发维护的，而且受到了Python官方推荐。Pipenv把虚拟环境管理有关的若干工具封装起来，提供了程序员友好的使用界面。这样，以后开发程序时就不再需要掌握那么多五花八门的工具，而只需要安装和学会用Pipenv就够了。

    安装Pipenv¶

    可考虑安装在全局Python环境里。

    1
    pip install pipenv

    创建虚拟环境¶

    进入工程目录的根目录，执行install命令（无参数），就可以自动为当前工程创建一个虚拟环境。

    1
2
    cd myproject
pipenv install

    创建好虚拟环境后，pipenv会在项目根目录下建立pipfile和pipfile.lock文件。之后，你就可以运行下面的命令检查和定位项目仓库及虚拟环境的位置。

    1
2
3
4
5
6
    $ pipenv --where
返回项目repository的目录
$ pipenv --venv
返回虚拟环境所在目录
$ pipenv --py
返回Python解释器所在目录

    激活虚拟环境¶

    进入工程目录或其子目录，执行shell命令就可以自动查找并激活当前工程目录所关联的虚拟环境。

    1
2
3
    cd myproject
pipenv shell
python --version

    安装依赖包¶

    可以用install命令安装指定版本的包，也可以不指定版本以安装最新版本的包。比如：

    1
2
3
    cd myproject
pipenv install requests==2.13.0
pipenv install requests

    也可以增加--dev表示开发环境所需的依赖包，没有指定--dev时默认按同时被开发环境和运行环境所依赖处理。

    1
2
    $ cd myproject
$ pipenv install --dev pytest

    更新依赖包¶

    可以用update命令升级所有的依赖包，也可以指定包名称。

    1
2
3
    cd myproject
pipenv update requests
pipenv update

    查看依赖包¶

    可以用graph命令查看当前项目的包依赖的关系图。

    1
2
3
4
5
6
    $ pipenv graph
requests==2.18.4
  - certifi [required: >=2017.4.17, installed: 2017.7.27.1]
  - chardet [required: >=3.0.2,<3.1.0, installed: 3.0.4]
  - idna [required: >=2.5,<2.7, installed: 2.6]
  - urllib3 [required: <1.23,>=1.21.1, installed: 1.22]

    卸载依赖包¶

    可以用uninstall命令卸载依赖包。

    1
2
    cd myproject
pipenv uninstall requests

    可以用-all参数卸载所有的包，清空当前的虚拟环境。

    1
2
    cd myproject
pipenv uninstall --all
