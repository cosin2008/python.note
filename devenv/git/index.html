    Git¶

    Git目前是最为流行的源码仓库，用于管理团队软件开发项目里的不同版本与分支的源代码。

    • Git不需要在开发过过程中与远程仓库始终保持联网
    • Git分支管理功能强大，创建分支的成本低、速度快
    • Git目前已经可以跨平台使用

    很有意思的是，Git最初是Linus在两周时间里开发出来的，取代其它版本控制系统来管理Linux核心代码

    基本概念和原理¶

    工作区、暂存区和分支¶

    每个git本地项目都被称为一个git仓库，仓库的结构如下：

    • 仓库（Repository）
    • 工作区（workspace）：所有的源码文件（可能会包括unstaged的修改）
    • 版本库（.git隐藏目录）：用来记录版本和分支信息的地方
      • 暂存区 (Stage)：在执行git commit签入存档前，所有标记过的改动（如git add）都被暂时保存在暂存区
      • 主分支（Master）：仓库初始化（git init）和从远程仓库（git clone）克隆到本地时默认创建的主分支
      • 其它分支：以本地分支或远程分支为基础创建（git branch）一个新分支
      • 当前版本（HEAD）：一个指向当前分支（git checkout）的头指针

    快照与修改历史¶

    为快速切换版本和分支，Git实际上并不会比较和保存两个版本的文件内容差异，而是用快照来实现。每签入一个新版本，实际上就是一次对暂存区里有变化的文件制作一个快照。这样，切换分支或版本时，只需要切换为特定的快照就可以了。

    修改历史实际上就是由上面的快照组成的一颗树，每个节点都是一个版本快照。分支就成了若干首尾连接的版本快照组成的修改历史树上的一条路径。

    如果想学习和了解更多关于Git的原理，建议阅读图解Git。

    Git客户端和远程仓库¶

    在Git网站上可以下载各个平台的Git客户端。

    如果需要多人合作、希望提供远程备份或者持续集成，那么你就需要考虑配合远程Git仓库使用。

    项目 github gitlab
    开源 是 是
    免费 是 是
    私有项目 收费 免费
    私有部署 不支持 免费
    稳定性 较好 一般

    简单的，对于个人来说：

    • 如果是私有Git项目或者需要独立部署Git仓库，请选择Gitlab
    • 如果是开源项目，请选择Github。

    常用Git命令¶

    基础命令¶

    命令 功能
    git init 把当前目录初始化为一个本地git仓库
    git init {project-name} 在当前目录下创建一个项目子目录并初始化为一个本地git仓库
    git add . 扫描当前目录（及子目录）下的所有修改，并记录到本地git仓库暂存区
    git add {file-name} 向本地git仓库暂存区添加一个或多个文件
    git add {path-name} 向本地git仓库暂存区添加一个目录（包括子目录和目录、子目录下的所有修改）
    git rm {file-name} 在本地git仓库暂存区增加对一个或多个文件的删除
    git mv {file-original} {file-renamed} 在本地git仓库暂存区增加对文件的重命名
    git commit 将当前暂存区修改快照写入本地git版本库存档

    本地分支管理命令¶

    命令 功能
    git branch {branch-name} 新建一个本地分支
    git branch 查看本地所有分支
    git checkout {branch-name} 切换当前分支
    git branch -d {branch-name} 删除一个本地分支

    撤销与重演¶

    命令 功能
    git revert {commit-id} 撤销特定的commit的修改（会新增一次Undo修改的commit）
    git revert HEAD 撤销最后一次commit的修改（会新增一次Undo修改的commit）
    git cherry-pick {commit-id} 反撤销已经被Undo的commit，相当于Redo
    git cherry-pick {commit-id} 选择其它分支特定commit的修改，将其重演到当前分支上

    合并冲突¶

    命令 功能
    git merge {branch-name} 将指定分支的所有修改合并到当前分支

    合并分支时有可能会出现冲突，发生冲突的时候需要手工修正冲突后commit。

    管理远程分支¶

    命令 功能
    git clone {remote-repo-url} 从远程仓库克隆一份到本地git仓库
    git branch -r 查看远程仓库里的所有远程分支
    git branch -d -r {branch-name} 删除一个远程分支
    git checkout -b {branch-name} origin/{remote-branch-name} 克隆远程分支，创建并切换到对应的本地分支
    git pull origin {remote-branch-name} 将远程分支的所有更新合并到当前本地分支（相当于git fetch加git merge）
    git fetch origin {remote-branch-name} 取回远程分支的所有更新，但不合并到当前本地分支（方便本地查看）
    git push origin {remote-branch-name} 把已签入的本地git仓库的修改版本推送到远端Git仓库（特定分支）
